# Git 명령어(상황별 종합 가이드)

**작성일**: Fri, 10 Oct 2025 17:34:32 +0900

**원문 링크**: https://junsseok.tistory.com/27

---

<p style="text-align: center;">아래는&nbsp;상황별로&nbsp;정리한&nbsp;실전&nbsp;가이드입니다. <br />필요한&nbsp;상황(예:&nbsp;깃허브에&nbsp;연결,&nbsp;충돌&nbsp;해결,&nbsp;커밋&nbsp;되돌리기&nbsp;등)을&nbsp;바로&nbsp;찾아서&nbsp;쓸&nbsp;수&nbsp;있게&nbsp;구성했습니다</p>
<hr contenteditable="false" />
<p style="text-align: left;">- 핵심 명령어(필수 암기!!)</p>
<pre class="bash" id="code_1760084324228"><code>(1) 새 로컬 저장소 초기화  
git init  

(2) 원격 저장소 복제  
git clone &lt;url&gt;  

(3) 상태 확인  
git status  

(4) 스테이징  
git add &lt;file|.&gt;  

(5) 커밋  
git commit -m "msg"  

(6) 원격에 푸시 (초기 업스트림 설정)  
git push -u origin main  

(7) 원격에서 가져와 병합  
git pull  

(8) 원격 가져오기 (병합 안 함)  
git fetch  

(9) 브랜치 목록 확인  
git branch  

(10) 브랜치 생성 + 전환  
git checkout -b feature/x  
# 또는  
git switch -c feature/x  

(11) 현재 브랜치에 병합  
git merge feature/x  

(12) 리베이스 (정리된 히스토리 유지)  
git rebase origin/main</code></pre>
<hr contenteditable="false" />
<p>1. 초기 설정 &amp; 저장소 생성 / 깃허브 연결</p>
<pre class="bash" id="code_1760084432651"><code>git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main</code></pre>
<p>&nbsp;</p>
<p>2. 새 프로젝트 시작해서 GitHub에 올리기</p>
<pre class="bash" id="code_1760084470307"><code>git init
git add .
git commit -m "커밋할 메세지"
git branch -M main
git remote add origin https://github.com/내아이디/레포이름.git
git push -u origin main</code></pre>
<p>&nbsp;</p>
<p>3. 브랜치(Branch)&nbsp;관리</p>
<p>(1) 브렌치 생성/전환</p>
<pre class="bash" id="code_1760084553421"><code>git checkout -b 생성할브렌치
git switch -c 전환할브렌치</code></pre>
<p>&nbsp;</p>
<p>(2) 모든 브렌치 보기</p>
<pre class="bash" id="code_1760084627507"><code>git branch  #로컬
git branch -a #로컬, 원격</code></pre>
<p>&nbsp;</p>
<p>(3) 브렌치 삭제</p>
<pre class="bash" id="code_1760084668850"><code>git branch -d name1   # 병합된 브랜치만 삭제
git branch -D name1   # 강제 삭제
git push origin --delete name1   # 원격 삭제</code></pre>
<p>&nbsp;</p>
<p>(4) 브렌치 pull</p>
<pre class="bash" id="code_1760084834498"><code>git checkout main
git merge name1</code></pre>
<p>&nbsp;</p>
<p>4. 원격(remote) &amp; 협업 워크플로</p>
<p>(1) 원격 확인/추가/삭제</p>
<pre class="bash" id="code_1760084755611"><code>git remote -v				 # 확인
git remote add origin &lt;url&gt;  # 추가
git remote remove origin	 # 삭제</code></pre>
<p>&nbsp;</p>
<p>(2) 원격 브랜치 가져오기</p>
<pre class="bash" id="code_1760084796362"><code>git fetch origin
git fetch --all</code></pre>
<p>&nbsp;</p>
<p>(3) 원격 pull</p>
<pre class="bash" id="code_1760084861562"><code>git pull</code></pre>
<p>&nbsp;</p>

<p>!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!! 자주 사용하는 명령어 !!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!</p>
<p>&nbsp;</p>
<p>(1) 깃허브에 push 할때&nbsp;</p>
<pre class="bash" id="code_1760084939250"><code>git add .
git commit -m "메세지"
git push origin main -&gt; 메인에 바로 푸시</code></pre>
<p>- 처음 한 번은 git push -u origin main 으로 업스트림 설정해야 함 <br />- 그다음부터는 git push만 입력해도 자동으로 main에 푸시됨</p>
<p>&nbsp;</p>
<p>(2) 깃허브에서 pull 할때&nbsp;</p>
<pre class="bash" id="code_1760085066722"><code>git pull origin main  # main에 있는 최신 내용 pull</code></pre>
<p>- 충돌(conflict) 생기면 파일 수정&nbsp; <br />- git add . -&gt; git commit 해서 해결</p>
<p>&nbsp;</p>
<p>(3) 새 프로젝트 시작할 때</p>
<pre class="bash" id="code_1760085117130"><code>git init
git add .
git commit -m "메세지"
git branch -M main
git remote add origin https://github.com/내아이디/레포이름.git
git push -u origin main</code></pre>
<p>&nbsp;</p>
<p>(4) 기존 프로젝트를 가져올때&nbsp;</p>
<pre class="bash" id="code_1760085132914"><code>git clone https://github.com/아이디/레포이름.git</code></pre>
<p>&nbsp;</p>
<p>(5)&nbsp;브랜치&nbsp;생성&nbsp;및&nbsp;전환할&nbsp;때 </p>
<pre class="bash" id="code_1760085198418"><code>git checkout -b 브랜치    # 브랜치 생성 + 전환
git switch -c 브랜치</code></pre>
