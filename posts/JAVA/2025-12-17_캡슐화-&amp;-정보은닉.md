<p style="text-align: center;">캡슐화&nbsp;&amp;&nbsp;정보은닉</p>

<p style="text-align: center;">자바의 접근제어자를 공부하던중 캡슐화와 정보은닉에 대해 알게됬어요. 다음과 같이 정리하며 공부해보려고 합니다.</p>
<p style="text-align: center;">&nbsp;</p>
<p style="text-align: center;"><b>캡슐화와 정보 은닉은 객체지향 프로그래밍(OOP)의 핵심 개념입니다.</b></p>
<p style="text-align: center;">&nbsp;</p>
<p style="text-align: center;">서로 밀접하지만 의미와 목적은 조금 달라요.</p>
<hr contenteditable="false" />
<h3 style="text-align: left;"><b>1. 캡슐화(Encapsulation)</b></h3>
<p>데이터(필드)와 그 데이터를 처리하는 메서드를 하나의 객체로 묶고, 외부에서는 정해진 방법으로만 접근하도록 설계하는 것이에요.</p>
<p>&nbsp;</p>
<p>-&gt; 간단하게 <b>객체의 내부 구현을 감싸서 하나의 단위로 만드는 것</b>입니다.</p>
<p>&nbsp;</p>
<p><b>설명</b><b></b></p>
<p style="color: #333333; text-align: start;">실생활로&nbsp;비유하자면&nbsp;자동차의&nbsp;운전자는&nbsp;엑셀,&nbsp;브레이크만&nbsp;사용합니다.&nbsp;</p>
<p style="color: #333333; text-align: start;">엔진&nbsp;내부&nbsp;동작은&nbsp;몰라도&nbsp;되죠.&nbsp;이렇게&nbsp;기능을&nbsp;하나를&nbsp;캡슐처럼&nbsp;묶어서&nbsp;제공해주는&nbsp;것입니다.</p>
<h3>목적</h3>
<p>(1) 객체 내부 구조를 몰라도 사용 가능</p>
<p><span style="color: #333333; text-align: start;">(2)<span> </span></span>코드 변경 시 영향 최소화</p>
<p><span style="color: #333333; text-align: start;">(3)<span> </span></span>객체의 책임과 역할을 명확히 함</p>
<p>&nbsp;</p>
<h3>예시 코드</h3>
<pre class="bash" id="code_1765960416636"><code>public class User {
    private String password;

    public void changePassword(String newPassword) {
        if (newPassword.length() &gt;= 8) {
            this.password = newPassword;
        }
    }
}</code></pre>
<p>&nbsp;</p>
<p>- password + changePassword() &rarr; 하나의 객체로 묶음 <br />- 비밀번호 변경 로직을 객체 내부에서 책임짐 <br /><br /><b>핵심&nbsp;포인트</b> <br />-&gt;&nbsp;데이터&nbsp;+&nbsp;행위&nbsp;=&nbsp;하나의&nbsp;객체</p>
<hr contenteditable="false" />
<h3 style="text-align: left;"><b>2. 정보 은닉(Information Hiding) </b></h3>
<p>&nbsp;</p>
<p>객체의 내부 데이터나 구현 세부사항을 외부에 숨기는 것입니다. 외부에서 알 필요 없는 정보는 숨기는 거죠.</p>
<p>&nbsp;</p>
<p><b>설명</b></p>
<p>실생활로 비유하면 ATM에서 사용자는 입,출금 버튼만 사용하죠.</p>
<p>여기서 은행 내부 전산 처리과정은 숨겨져요. 정보를 은닉하는 것입니다.<b><br /></b></p>
<h3>목적</h3>
<p><span style="color: #333333; text-align: start;">(1)<span>&nbsp;</span></span> 잘못된 접근 방지</p>
<p><span style="color: #333333; text-align: start;">(2)<span>&nbsp;</span></span> 데이터 무결성 보장</p>
<p><span style="color: #333333; text-align: start;">(3)<span>&nbsp;</span></span> 보안 강화</p>
<p>&nbsp;</p>
<h3>예시 코드</h3>
<pre class="bash" id="code_1765960697123"><code>public class Account {
    private int balance;

    public int getBalance() {
        return balance;
    }

    public void deposit(int amount) {
        if (amount &gt; 0) {
            balance += amount;
        }
    }
}</code></pre>
<p>&nbsp;</p>
<p>- private, protected 접근제어자 사용</p>
<p>- getter/setter로 제한적 접근 허용</p>
<p>- balance 직접 접근하지 않고 검증 로직을 거친 메서드로만 접근</p>

<h3 style="text-align: center;">면접한줄요약</h3>
<p style="text-align: center;"><b>캡슐화는&nbsp;객체의&nbsp;데이터와&nbsp;기능을&nbsp;하나로&nbsp;묶어&nbsp;책임을&nbsp;분리하는&nbsp;것이고,&nbsp;</b></p>
<p style="text-align: center;"><b>정보&nbsp;은닉은&nbsp;객체&nbsp;내부&nbsp;구현을&nbsp;숨기고&nbsp;필요한&nbsp;부분만&nbsp;공개하는&nbsp;것입니다.</b></p>
<p style="text-align: center;"><b>따라서 캡슐화는 구조, 정보 은닉은 접근 제어에 초점을 둔 개념입니다.</b></p>
