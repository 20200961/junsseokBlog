<div>
<div>
<p style="text-align: center;">자바로 코딩하다 보면 public, private 이런 키워드들이 자주 보입니다. 처음엔 이거 왜 쓰는 거지?</p>
<p style="text-align: center;">싶었는데, 알고 보니 코드 설계에서 엄청 중요한 역할을 합니다.</p>
<hr contenteditable="false" /></div>
</div>
<h3 style="text-align: center;">접근제어자</h3>
<p>&nbsp;</p>
<p>접근제어자는 말 그대로 클래스, 메서드, 변수 같은 것들에 대한 접근 권한을 제어하는 키워드예요.</p>
<p>쉽게 말해서 '이건 누가 볼 수 있고, 이건 숨길 거야'를 정하는 거죠. 이에 따라 정보은닉과 캡슐화 개념을 알아야 합니다.</p>
<p>&nbsp;</p>
<div>
<div>
<h3 style="text-align: center;">정보 은닉과 캡슐화?</h3>
<p>&nbsp;</p>
<p><b>정보 은닉(Information Hiding)</b>: 객체의 내부 구현을 외부로부터 감추는 것. 스마트폰 쓸 때 내부 회로를 몰라도 되는 것처럼요.</p>
<p><b>캡슐화(Encapsulation)</b>: 관련된 데이터와 메서드를 하나로 묶고, 외부 접근을 제어하는 것. 약 캡슐처럼 내용물을 감싸는 거죠.</p>
</div>
</div>
<div>
<pre class="bash" id="code_1765439272555"><code>public class BankAccount {
    // 캡슐화: 관련 데이터를 하나로 묶음
    private int balance;  // 정보 은닉: 외부에서 직접 접근 불가
    
    // 안전한 방법으로만 접근 허용
    public void deposit(int amount) {
        if (amount &gt; 0) {
            balance += amount;
        }
    }
}</code></pre>
<p>&nbsp;</p>
</div>
<p>자바에는 4가지 종류가 있습니다.</p>
<div>
<div><hr contenteditable="false" /></div>
</div>
<h4>1. public - 모두에게 오픈</h4>
<p>public은 가장 개방적인 접근제어자입니다. 어디서든 접근이 가능해요.</p>
<pre class="bash" id="code_1765438171812"><code>// 카페 메뉴판 - 누구나 볼 수 있음
public class CafeMenu {
    public String menuName = "아메리카노";
    public int price = 4500;
    
    public void showMenu() {
        System.out.println(menuName + ": " + price + "원");
    }
}

// 다른 패키지에서도 사용 가능
public class Customer {
    public void order() {
        CafeMenu menu = new CafeMenu();
        menu.showMenu();  // 어디서든 접근 OK!
    }
}</code></pre>
<p>&nbsp;</p>
<p>-&gt;&nbsp; 다른 패키지에서도, 어디서든 이 클래스와 메서드를 사용할 수 있어요.</p>
<p>&nbsp; &nbsp; &nbsp; API를 만들 때 외부에 공개할 기능들은 보통 public으로 선언합니다.</p>
<hr contenteditable="false" />
<h4>2. private - 나만 볼 수 있어</h4>
<p>private은 같은 클래스 내부에서만 접근 가능해요.</p>
<p><b>!!!! -----&gt;<span>&nbsp;</span>정보 은닉을 구현하는 핵심 접근제어자</b></p>
<pre class="bash" id="code_1765438252995"><code>// 은행 계좌 - 잔액은 외부에서 직접 못 건드림
public class BankAccount {
    private int balance = 100000;  // 외부에서 직접 접근 불가
    
    // 메서드를 통해서만 접근 가능
    public void deposit(int amount) {
        if (amount &gt; 0) {
            balance += amount;
        }
    }
    
    public int getBalance() {
        return balance;
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount();
        // account.balance = -999999;  // 에러! private라 접근 불가
        account.deposit(50000);  // 메서드로만 가능
        System.out.println(account.getBalance());
    }
}</code></pre>
<p>&nbsp;</p>
<p>- &gt; 민감한 정보나 내부에서만 사용하는 헬퍼 메서드들은 private으로 숨기는 게 좋아요. <b>캡슐화</b>의 핵심이에요.</p>
<hr contenteditable="false" />
<h4>3. protected - 가족끼리만</h4>
<p>protected는 같은 패키지 내에서, 그리고 다른 패키지더라도 상속받은 자식 클래스에서는 접근할 수 있습니다.</p>
<pre class="bash" id="code_1765438400372"><code>// 동물 부모 클래스
public class Animal {
    protected String name;
    protected int age;
    
    protected void sleep() {
        System.out.println(name + "이(가) 잠을 잡니다.");
    }
}

// 자식 클래스 - 다른 패키지에 있어도 OK
public class Dog extends Animal {
    public void introduce() {
        name = "뽀삐";  // 부모의 protected 멤버 사용 가능
        age = 3;
        System.out.println("이름: " + name + ", 나이: " + age);
        sleep();  // 부모의 protected 메서드 호출 가능
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.introduce();  // OK
        // dog.sleep();  // 에러는 아니지만 같은 패키지가 아니면 직접 호출 못함
    }
}</code></pre>
<p>&nbsp;</p>
<p>-&gt; 상속 구조에서 자식 클래스한테는 보여줘야 하는데, 완전히 public으로 열기는 싫을 때 쓰면 딱이에요.</p>
<p>&nbsp;</p>
<p style="text-align: center;"><b>상속 관계에서 캡슐화를 유지하면서도 자식 클래스에 필요한 부분만 공개할 때 사용해요!</b></p>
<p>&nbsp;</p>
<hr contenteditable="false" />
<h4>4. default (package-private) - 같은 동네끼리만</h4>
<p>아무것도 안 쓰면 default 접근제어자가 적용됩니다. 같은 패키지 내에서만 접근 가능해요.</p>
<pre class="bash" id="code_1765438503707"><code>// 같은 패키지 내의 유틸리티 클래스
class Calculator {  // public 안 붙임 = default
    int add(int a, int b) {  // default 메서드
        return a + b;
    }
}

// 같은 패키지에서는 사용 가능
public class MathTest {
    public void test() {
        Calculator calc = new Calculator();
        System.out.println(calc.add(1, 2));  // OK
    }
}</code></pre>
<p>&nbsp;</p>
<p>-&gt; 사실 실무에서는 명시적으로 쓰는 게 좋아서 default는 잘 안 쓰게 됩니다!</p>
<hr contenteditable="false" />
<h3 style="text-align: center;">&nbsp;</h3>
<h2>정리하면</h2>
<p>접근 범위를 넓은 순서대로 보면 이렇습니다: <b>public &gt; protected &gt; default &gt; private</b></p>
<p><figure class="imageblock alignCenter"><span><img height="223" src="https://blog.kakaocdn.net/dn/bc0GWF/dJMcaa4SF6t/KVQKLlks39UTUA7HjmMJxK/img.png" width="718" /></span></figure>
</p>
<p>&nbsp;</p>
<p>"최소한의 접근 권한만 주자"는 원칙이 중요해요. 처음엔 다 public으로 해놨다가 나중에 수정하려니 다른 곳에서 막 쓰고 있어서 난감했던 경험이... ㅠㅠ</p>
<p>&nbsp;</p>
<p>기본적으로는 private으로 시작해서, 필요할 때만 접근 범위를 넓히는 게 좋은 습관입니다. 이렇게 하면 나중에 코드 수정할 때도 영향 범위를 파악하기 쉽고, 버그도 줄일 수 있어요.</p>
<p>&nbsp;</p>
<p>접근제어자를 잘 활용하면 훨씬 안전하고 유지보수하기 좋은 코드를 만들 수 있어요!</p>
