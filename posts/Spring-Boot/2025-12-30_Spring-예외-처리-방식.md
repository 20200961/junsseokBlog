<p style="text-align: center;">Spring&nbsp;예외처리방식</p>
<hr contenteditable="false" />
<p style="text-align: center;">예외 처리는 서비스 안정성, 가독성, 유지보수성에 직결되는 핵심 요소입니다.</p>
<p style="text-align: center;">Spring Framework는 애플리케이션에서 발생하는 예외를 효과적으로 관리할 수 있는 다양한 방법을 제공합니다.&nbsp;</p>
<hr contenteditable="false" />
<h2>예외(Exception)란?</h2>
<p>예외란 프로그램 실행 중 발생하는 비정상적인 상황을 의미합니다.</p>
<p>예시:</p>
<p>- 잘못된 요청 파라미터</p>
<p>- DB 조회 결과 없음</p>
<p>- NullPointerException</p>
<p>- 권한 없는 접근</p>
<p>Spring은 이런 예외를 컨트롤러 -&gt; 서버 전체 단위로 관리할 수 있도록 다양한 예외 처리 방식을 제공합니다.</p>
<hr contenteditable="false" />
<h3>Spring MVC의 예외 처리 흐름</h3>
<pre class="bash" id="code_1767083931481"><code>클라이언트 요청
   &darr;
Controller
   &darr;
Service
   &darr;
Repository
   &darr;
예외 발생
   &darr;
Exception Handler
   &darr;
HTTP 응답 반환</code></pre>
<p>Spring은 예외가 발생하면 DispatcherServlet이 가로채서 적절한 예외 처리 로직으로 전달합니다.</p>
<hr contenteditable="false" />
<h3 style="text-align: left;">@ExceptionHandler - 컨트롤러 단위 처리</h3>
<p>컨트롤러 내에서 발생하는 특정 예외를 처리하는 가장 기본적인 방법입니다.</p>
<p>컨트롤러 클래스 내부에 예외 처리 메서드를 정의하여 해당 컨트롤러에서 발생한 예외를 처리합니다.</p>
<pre class="bash" id="code_1767082424226"><code>@RestController
public class UserController {
    
    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity&lt;String&gt; handleUserNotFound(UserNotFoundException ex) {
        return ResponseEntity.status(404).body(ex.getMessage());
    }
}</code></pre>
<p>&nbsp;</p>
<p><b>사용 시기 : </b>특정 컨트롤러에서만 발생하는 예외를 처리하거나, 해당 컨트롤러만의 특별한 예외 처리 로직이 필요할 때 사용합니다.</p>
<p>하지만 여러 컨트롤러에서 같은 예외 처리가 필요하면 코드 중복이 발생한다는 단점이 있습니다.</p>
<p>&nbsp;</p>
<p><b>사용 예 :</b> 결제 컨트롤러에서 결제 취소 예외가 발생하면 환불 처리를 함께 진행해야 하는 경우, 또는 관리자 컨트롤러에서는 상세한 에러 정보를 보여주고 일반 사용자 컨트롤러에서는 간단한 메시지만 보여줘야 하는 경우에 적합합니다. 하지만 여러 컨트롤러에서 같은 예외 처리가 필요하면 코드 중복이 발생한다는 단점이 있습니다.</p>
<hr contenteditable="false" />
<h3>@ControllerAdvice - 전역 예외 처리</h3>
<p>애플리케이션 전체에서 발생하는 예외를 한 곳에서 처리합니다.</p>
<p>별도의 예외 처리 클래스를 만들어 모든 컨트롤러의 예외를 중앙에서 관리합니다.</p>
<pre class="bash" id="code_1767082467402"><code>@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity&lt;ErrorResponse&gt; handleUserNotFound(UserNotFoundException ex) {
        return ResponseEntity.status(404)
            .body(new ErrorResponse("USER_NOT_FOUND", ex.getMessage()));
    }
    
    @ExceptionHandler(Exception.class)
    public ResponseEntity&lt;ErrorResponse&gt; handleGeneral(Exception ex) {
        return ResponseEntity.status(500)
            .body(new ErrorResponse("SERVER_ERROR", "서버 오류가 발생했습니다"));
    }
}</code></pre>
<p>&nbsp;</p>
<p><b>사용 시기 : </b>실무에서 가장 많이 사용되는 방식으로, API 전체에서 일관된 에러 응답 형식을 유지해야 할 때 사용합니다.</p>
<p>&nbsp;</p>
<p><b>사용 예 :</b><span style="color: #333333; text-align: start;"><span> 모든 컨트롤러에서 발생하는 유효성 검증 실패를 동일한 형식으로 처리하거나, 인증/인가 실패를 공통으로 처리해야 하는 경우에 적합합니다. </span></span></p>
<hr contenteditable="false" />
<h3>@RestControllerAdvice (REST API 전용)</h3>
<p>REST API에서는 View가 아닌 JSON 응답이 필요합니다.</p>
<pre class="bash" id="code_1767084018745"><code>@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(IllegalArgumentException.class)
    public ResponseEntity&lt;String&gt; handleIllegalArgument(Exception e) {
        return ResponseEntity
                .badRequest()
                .body(e.getMessage());
    }
}</code></pre>
<p>&nbsp;</p>

<h3>@ResponseStatus - 예외 클래스에 직접 지정</h3>
<p>커스텀 예외 클래스를 만들 때 HTTP 상태 코드를 직접 명시하는 방법입니다.</p>
<p>예외 클래스 자체에 어노테이션을 붙여서 해당 예외가 발생하면 자동으로 지정된 상태 코드가 반환됩니다.</p>
<pre class="bash" id="code_1767082992473"><code>@ResponseStatus(HttpStatus.NOT_FOUND)
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}</code></pre>
<p>&nbsp;</p>
<p style="color: #333333; text-align: start;"><b>사용 시기 :<span>&nbsp;</span></b> 간단한 예외 처리가 필요하고 예외와 HTTP 상태 코드의 매핑이 명확할 때 사용합니다.</p>
<p style="color: #333333; text-align: start;">&nbsp;</p>
<p style="color: #333333; text-align: start;"><b>사용 예 :</b><span style="color: #333333; text-align: start;"><span> 리소스를 찾을 수 없는 경우는 항상 404를, 권한이 없는 경우는 항상 403을 반환하는 등 </span></span></p>
<p style="color: #333333; text-align: start;"><span style="color: #333333; text-align: start;"><span>예외와 상태 코드가 1:1로 대응될 때 적합합니다.</span></span></p>
<hr contenteditable="false" />
<h3 style="text-align: center;">면접한줄요약</h3>
<p style="text-align: center;"><b> Spring 예외 처리는 컨트롤러에서 직접 처리하지 않고, 전역 예외 처리기를 통해 공통으로 관리하는 방식입니다. </b></p>
