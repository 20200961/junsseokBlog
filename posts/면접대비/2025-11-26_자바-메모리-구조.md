<hr contenteditable="false" />

<p style="text-align: center;">&nbsp;</p>
<p style="text-align: center;"><b>Method area, Stack area, Heap area, Pc register area, Native method area</b></p>
<hr contenteditable="false" />
<ol>
<li>Method Area</li>
</ol>
<ul>
<li>역할 :<br />-- 클래스 단위 정보 저장<br />-- 프로그램 시작 시 클래스가 로딩될 때 생성</li>
<li>저장내용 :<br />-- 클래스 메타데이터(클래스 이름, 상속 정보)<br />-- 메서드 바이트코드<br />-- static 변수<br />-- 상수 풀(Constant Pool)</li>
<li>특징 :<br />-- Java 8 이후 PermGen &rarr; Metaspace<br />-- Heap이 아닌 Native Memory 사용<br />-- GC 대상<br />-- 클래스 언로드 시 메모리 해제</li>
</ul>
<hr contenteditable="false" />
<p>2. Heap</p>
<ul>
<li>역할 :<br />-- new 키워드로 생성된 객체 저장<br />-- 모든 스레드가 공유</li>
<li>구조 :<br />-- Young Generation<br />--- Eden<br />--- Survivor(S0, S1)<br />-- Old Generation</li>
<li>특징 :<br />-- Young Generation에서 Minor GC 발생<br />-- Old Generation에서 Major GC / Full GC 발생<br />-- GC가 자동으로 메모리 관리<br />-- 메모리 부족 시 OutOfMemoryError 발생</li>
</ul>
<hr contenteditable="false" />
<p>3. Stack</p>
<ul>
<li>역할 :<br />-- 메서드 호출과 지역 변수 관리<br />-- 스레드마다 독립적으로 생성</li>
<li>저장내용 :<br />-- 지역 변수<br />-- 매개변수<br />-- 리턴 주소<br />-- 참조 변수(객체 실제 데이터는 Heap에 저장)</li>
<li>특징 :<br />-- 메서드 호출 시 Stack Frame 생성<br />-- 메서드 종료 시 자동 제거<br />-- 메모리 접근 속도 빠름<br />-- StackOverflowError 발생 가능</li>
</ul>
<hr contenteditable="false" />
<p>4. PC Register</p>
<ul>
<li>역할 :<br />-- 현재 실행 중인 JVM 명령어 주소 저장</li>
<li>특징 :<br />-- 스레드마다 하나씩 존재<br />-- 스레드 전환 시 실행 위치 유지</li>
</ul>
<hr contenteditable="false" />
<p>5. Native Method Stack</p>
<ul>
<li>역할 :<br />-- C/C++ 등 Native 메서드(JNI) 실행 시 사용</li>
<li>특징 :<br />-- 자바 스택과 구조 유사<br />-- 플랫폼(JVM 구현)에 의존적</li>
</ul>
