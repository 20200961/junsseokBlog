<p style="text-align: center;">servlet 및 개념 정리</p>
<hr contenteditable="false" />
<p>1. Servlet이란? <br /><br />자바&nbsp;기반의&nbsp;웹&nbsp;기술로,&nbsp;클라이언트의&nbsp;요청(Request)을&nbsp;받아서&nbsp;처리한&nbsp;뒤&nbsp;응답(Response)을&nbsp;만들어주는&nbsp;자바&nbsp;클래스. <br />사용자가&nbsp;입력한&nbsp;값을&nbsp;서버로&nbsp;보내면&nbsp;서블릿이&nbsp;이를&nbsp;처리하고,&nbsp;결과를&nbsp;다시&nbsp;HTML/JSP&nbsp;화면으로&nbsp;보여줄&nbsp;수&nbsp;있다. <br /><br />2.&nbsp;GET&nbsp;방식&nbsp;요청&nbsp;처리 <br />(1)&nbsp;특징 <br />- URL에 데이터를 포함해서 전달 (?name=값&amp;age=값 형태)<br />- 보안에 취약 (데이터가 URL에 그대로 노출)<br />- 전송 길이 제한(2000자)<br />- 캐싱/즐겨찾기 가능 &rarr; 검색, 조회에 적합<br /><br /></p>
<p>HTML(form)</p>
<pre class="bash" id="code_1758878531161"><code>&lt;form action="/servlet/gettest.do" method="get"&gt;
    이름 : &lt;input type="text" name="userName"&gt;
    나이 : &lt;input type="number" name="age"&gt;
    성별 : 
    &lt;input type="radio" name="gender" value="M"&gt; 남자
    &lt;input type="radio" name="gender" value="F"&gt; 여자
    &lt;input type="submit"&gt;
&lt;/form&gt;</code></pre>
<p>&nbsp;</p>
<p>Servlet(RequestGetServlet.java)</p>
<pre class="bash" id="code_1758878570463"><code>@WebServlet("/gettest.do")
public class RequestGetServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        String name = request.getParameter("userName");
        int age = Integer.parseInt(request.getParameter("age"));
        String gender = request.getParameter("gender");

        // 응답 페이지를 직접 생성
        response.setContentType("text/html; charset=utf-8");
        PrintWriter out = response.getWriter();
        
        out.println("&lt;html&gt;&lt;body&gt;");
        out.printf("&lt;h2&gt;%s님의 정보&lt;/h2&gt;", name);
        out.printf("&lt;p&gt;나이: %d&lt;/p&gt;", age);
        out.printf("&lt;p&gt;성별: %s&lt;/p&gt;", gender);
        out.println("&lt;/body&gt;&lt;/html&gt;");
    }
}</code></pre>
<p>&nbsp;</p>
<p>-&gt; GET 방식 서블릿은 요청 값을 꺼내서 바로 HTML을 작성하여 응답한다</p>
<p>&nbsp;</p>
<p>&nbsp;</p>
<p>3.&nbsp;POST&nbsp;방식&nbsp;요청&nbsp;처리 <br />(1)&nbsp;특징 <br />-&nbsp;HTTP&nbsp;Body&nbsp;영역에&nbsp;데이터를&nbsp;담아&nbsp;전송. <br />-&nbsp;URL에&nbsp;노출되지&nbsp;않아&nbsp;보안에&nbsp;유리. <br />-&nbsp;전송&nbsp;길이&nbsp;제한&nbsp;없음&nbsp;&rarr;&nbsp;회원가입,&nbsp;글쓰기&nbsp;적합. <br />-&nbsp;즐겨찾기&nbsp;가능하지만&nbsp;데이터는&nbsp;저장되지&nbsp;않음.</p>
<p>&nbsp;</p>
<p>HTML&nbsp;(form)</p>
<pre class="bash" id="code_1758880725425"><code>&lt;form action="/servlet/posttest.do" method="post"&gt;
    이름 : &lt;input type="text" name="userName"&gt;
    나이 : &lt;input type="number" name="age"&gt;
    성별 : 
    &lt;input type="radio" name="gender" value="M"&gt; 남자
    &lt;input type="radio" name="gender" value="F"&gt; 여자
    &lt;input type="submit"&gt;
&lt;/form&gt;</code></pre>
<p>&nbsp;</p>
<p>Servlet&nbsp;(RequestPostServlet.java)</p>
<pre class="bash" id="code_1758880739465"><code>@WebServlet("/posttest.do")
public class RequestPostServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        // 요청값 추출
        String name = request.getParameter("userName");
        int age = Integer.parseInt(request.getParameter("age"));
        String gender = request.getParameter("gender");

        // request 객체에 데이터 저장 (JSP로 전달하기 위함)
        request.setAttribute("name", name);
        request.setAttribute("age", age);
        request.setAttribute("gender", gender);

        // JSP로 포워딩
        RequestDispatcher view = request.getRequestDispatcher("/views/responsePage.jsp");
        view.forward(request, response);
    }
}</code></pre>
<p>&nbsp;</p>

<pre class="bash" id="code_1758880498060"><code>&lt;%

&lt;h2&gt;개인정보 응답화면&lt;/h2&gt;
&lt;p&gt;&lt;%=name %&gt;님은 &lt;%=age %&gt;살이며 성별은 &lt;%=gender %&gt;입니다.&lt;/p&gt;</code></pre>
<p>-&gt; POST&nbsp;방식&nbsp;서블릿은&nbsp;request.setAttribute()&nbsp;로&nbsp;값을&nbsp;저장한&nbsp;뒤&nbsp;JSP에&nbsp;전달하여&nbsp;템플릿을&nbsp;통해&nbsp;응답&nbsp;화면을&nbsp;구성한다.</p>
<p>&nbsp;</p>
<p>4.&nbsp;핵심&nbsp; <br />GET&nbsp;-&gt;&nbsp;요청&nbsp;데이터를&nbsp;URL로&nbsp;전달,&nbsp;조회용 <br />POST&nbsp;-&gt;&nbsp;요청&nbsp;데이터를&nbsp;Body에&nbsp;담아&nbsp;전달,&nbsp;중요한&nbsp;데이터&nbsp;처리용 <br /><br />응답&nbsp;처리&nbsp;방식&nbsp;:&nbsp; <br />RequestGetServlet:&nbsp;HTML을&nbsp;자바&nbsp;코드에서&nbsp;직접&nbsp;작성 <br />RequestPostServlet:&nbsp;JSP로&nbsp;포워딩해서&nbsp;뷰를&nbsp;분리</p>
<hr contenteditable="false" />
<p>배운점 &amp; 느낀점</p>
<p>Servlet을&nbsp;학습하면서&nbsp;웹에서&nbsp;요청과&nbsp;응답이&nbsp;어떻게&nbsp;처리되는지&nbsp;흐름을&nbsp;이해할&nbsp;수&nbsp;있었고&nbsp;GET과&nbsp;POST&nbsp;방식의&nbsp;차이도&nbsp;확실히&nbsp;알게&nbsp;되었다.&nbsp;GET은&nbsp;URL에&nbsp;데이터를&nbsp;담아&nbsp;간단한&nbsp;조회에&nbsp;적합하지만&nbsp;보안에&nbsp;취약하고&nbsp;길이&nbsp;제한이&nbsp;있다는&nbsp;점과&nbsp;POST는&nbsp;데이터를&nbsp;Body에&nbsp;담아&nbsp;전송하기&nbsp;때문에&nbsp;보안성과&nbsp;유연성이&nbsp;높아&nbsp;중요한&nbsp;데이터&nbsp;처리에&nbsp;적합하다는&nbsp;점이&nbsp;인상&nbsp;깊었다.&nbsp;또한&nbsp;GET은&nbsp;서블릿&nbsp;안에서&nbsp;직접&nbsp;HTML을&nbsp;작성해&nbsp;응답하는&nbsp;방식이고&nbsp;POST는&nbsp;JSP로&nbsp;데이터를&nbsp;넘겨&nbsp;화면을&nbsp;구성함으로써&nbsp;로직과&nbsp;뷰를&nbsp;분리할&nbsp;수&nbsp;있어&nbsp;구조적인&nbsp;측면에서&nbsp;더&nbsp;깔끔하다는&nbsp;것도&nbsp;배웠다.&nbsp;이를&nbsp;통해&nbsp;단순히&nbsp;동작하는&nbsp;코드를&nbsp;작성하는&nbsp;것을&nbsp;넘어&nbsp;상황에&nbsp;맞는&nbsp;요청&nbsp;방식을&nbsp;선택하고&nbsp;유지보수성을&nbsp;고려한&nbsp;구조를&nbsp;설계하는&nbsp;것이&nbsp;중요하다는&nbsp;것을&nbsp;느꼈다.</p>
