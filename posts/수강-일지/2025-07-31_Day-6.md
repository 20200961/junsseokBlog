<p>오늘은 Oracle SQL 수업에서 DML(Data Manipulation Language)과 DDL(Data Definition Language)에 대해 학습하고, 실습을 통해 각 명령어의 사용 방법을 익혔다.</p>
<p>&nbsp;</p>
<p>1. DML (데이터 조작어)</p>
<p>(1) INSERT : 테이블에&nbsp;새로운&nbsp;행(ROW)를&nbsp;추가하는&nbsp;구문</p>
<pre class="bash" id="code_1753949793686"><code>INSERT INTO 테이블명 VALUES(값,값,값...)</code></pre>
<p>테이블의 모든 컬럼에 값을 순서대로 삽입할 때 사용</p>
<p>VALUES에&nbsp;직접&nbsp;값을&nbsp;명시하는&nbsp;것이&nbsp;아니라,&nbsp;대신&nbsp;서브쿼리로&nbsp;조회된&nbsp;값을&nbsp;모두&nbsp;INSERT&nbsp;가능</p>
<p>테이블의 모든 컬럼에 대한 값을 순서대로 제시해서 한 행을 INSERT하고자 할 때 사용</p>
<p>컬럼의 수와 순번을 지켜서 VALUES 값을 나열해야 한다.</p>
<pre class="bash" id="code_1753949819125"><code>INSERT INTO 테이블명(컬럼1, 컬럼2, ...) VALUES(...)</code></pre>
<p>필요한 컬럼만 선택하여 삽입 가능</p>
<pre class="bash" id="code_1753949835357"><code>INSERT INTO 테이블명 (서브쿼리)</code></pre>
<p>서브쿼리로부터 다중 행을 한 번에 삽입</p>
<p>&nbsp;</p>
<p>(2) INSERT ALL(다중&nbsp;테이블&nbsp;삽입) <br />&nbsp;&nbsp;&nbsp;&nbsp;두개&nbsp;이상의&nbsp;테이블에&nbsp;각각&nbsp;INSERT할&nbsp;때&nbsp;사용되는&nbsp;서브쿼리가&nbsp;동일한&nbsp;경우</p>
<pre class="bash" id="code_1753949977741"><code>INSERT ALL
    INTO 테이블명1 VALUES(컬럼,컬럼,컬럼...)
    INTO 테이블명2 VALUES(컬럼,컬럼...)
    서브쿼리;</code></pre>
<p>&nbsp;</p>
<p>(3) UPDATE : 테이블에&nbsp;기록되어&nbsp;있는&nbsp;기존의&nbsp;데이터를&nbsp;수정하는&nbsp;구문</p>
<p>!! WHERE 절을 생략하면 전체 행이 수정되므로 주의</p>
<pre class="bash" id="code_1753950062533"><code>    UPDATE 테이블명
    SET 컬럼1 = 값1
        컬럼2 = 값2
    [WHERE 조건];</code></pre>
<p>WHERE절이 없으면 전체 행 수정</p>
<p>UPDATE시에도 제약조건을 잘 확인해야한다</p>
<p>&nbsp;</p>
<p>(4) DELETE : 테이블에 기록된 데이터를 삭제하는 구문(한 행 단위로 삭제가 됨)</p>
<pre class="bash" id="code_1753950197493"><code>DELETE FROM 테이블명
    [WHERE 조건]</code></pre>
<p>!! 조건을 입력하지 않으면 전체행이 삭제</p>
<p>&nbsp;</p>
<p>2. DDL (데이터 제어문)</p>
<p>계정에 시스템권한 또는 객체 접근 권한을 부여하거나 회수하는 구문</p>
<p>즉, 누가 어떤 객체에 무엇을 할 수 있는지를 통제함.</p>
<p>&nbsp;</p>
<p>&gt;&nbsp;시스템&nbsp;권한&nbsp;:&nbsp;DB객체생성,&nbsp;사용자&nbsp;생성&nbsp;등&nbsp;관리자권한 <br />&gt; 객체 접근권한 : 테이블, 뷰, 시퀀스 등 특정 객체에 접근 권한</p>
<p>&nbsp;</p>
<p>1) 사용자 생성</p>
<pre class="bash" id="code_1753950907724"><code>CREATE USER 사용자명 IDENTIFIED BY 비밀번호;;</code></pre>
<p>&nbsp;</p>
<p>2)&nbsp;권한&nbsp;부여</p>
<pre class="bash" id="code_1753950930972"><code>GRANT 권한(RESOURCE, CONNECT) TO 계정;</code></pre>
<p>&nbsp;</p>
<p>3) 권한 회수</p>
<pre class="bash" id="code_1753950942404"><code>REBOKE 권한 TO 계정;</code></pre>
<p>&nbsp;</p>
<p>ORACLE은&nbsp;기본적으로&nbsp;사용자&nbsp;생성&nbsp;후&nbsp;권한을&nbsp;부여해야&nbsp;로그인&nbsp;및&nbsp;객체&nbsp;사용&nbsp;가능</p>
<p>&nbsp;</p>
<p>3. &nbsp;TCL : 트랜잭션 제어문</p>
<p>트랜잭션은 하나 이상의 DML(INSERT,UPDATE,DELETE)문을 묶어서 하나의 논리적 작업 단위로 처리하는 개념</p>
<pre class="bash" id="code_1753951010477"><code>DML문 한개를 수행할 때 트랜잭션이 존재하지 않으면 트랜잭션을 만들어서 묶음.
트랜잭션이 존재하면 해당 트랜잭션에 직업을 묶어서 처리.
COMMIT : 트랜잭션 확정 및 저장
ROLLBACK : 트랜잭션 취소 및 이전 상태로 복원
    
SAVEPOINT : 롤백용으로 중간지점을 지정
ROLLBACK TO : 특정 지점까지 복원</code></pre>
<p>&nbsp;</p>
<p>대부분&nbsp;요즘은&nbsp;AUTOCOMMIT이&nbsp;설정되어&nbsp;있어서&nbsp;DML을&nbsp;전달할&nbsp;때마다&nbsp;자동으로&nbsp;커밋이&nbsp;됨</p>
<p>&nbsp;</p>
<p>&nbsp;</p>
<p>4. DDL문(CREATE,ALTER,DROP)</p>
<p>수행하는 순간 기존 트랜잭션은 무조건 COMMIT됨(실제 DB반영) <br />즉, DDL문 수행 전 변경사항들이 있다면 정확하게 픽스하고 해라.</p>
<pre class="bash" id="code_1753951087325"><code>ACID 속성 
트랜잭션의 일관성과 신뢰성을 보장하기 위해 지켜야하는 4가지 핵심 속성
    
A(AUTOMICITY) 원자성 : 트랜잭션 내 작업은 모두 완료되거나, 모두 취소되어야 한다.
C(CONSISTENCY) 일관성 : 트랜잭션 수행 전후에 DB상태가 항상 유효한 상태(제약조건 충족)여야 한다.
I(ISOLATION) 독립성 : 여러 트랜잭션이 동시에 실행되도 서로 간섭없이 독립적으로 실행되야한다.
D(DURABILITY) 지속성 : 트랜잭션이 COMMIT 되면, 그 결과는 시스템이 종료되고 영구적으로 보존되어야 한다.</code></pre>
<p>&nbsp;</p>
<p>5. 배운점 &amp; 느낀점&nbsp;</p>
<p>이번&nbsp;수업을&nbsp;통해&nbsp;단순히&nbsp;데이터를&nbsp;조회하는&nbsp;것(SELECT)에만&nbsp;그치지&nbsp;않고,&nbsp;INSERT,&nbsp;UPDATE,&nbsp;DELETE를&nbsp;활용해&nbsp;실제로&nbsp;데이터를&nbsp;삽입하고&nbsp;수정&middot;삭제하는&nbsp;과정까지&nbsp;실습해볼&nbsp;수&nbsp;있었다. <br /><br />특히&nbsp;UPDATE&nbsp;구문에서&nbsp;서브쿼리를&nbsp;활용해&nbsp;다른&nbsp;사원의&nbsp;값을&nbsp;참조하여&nbsp;수정하는&nbsp;방식이나,&nbsp;INSERT&nbsp;ALL을&nbsp;통해&nbsp;여러&nbsp;테이블에&nbsp;동시에&nbsp;데이터를&nbsp;삽입하는&nbsp;고급&nbsp;문법은&nbsp;실제&nbsp;현업에서도&nbsp;활용도가&nbsp;높을&nbsp;것&nbsp;같아&nbsp;기억에&nbsp;남는다. <br /><br />또한 ALTER, DROP 등 DDL 문법을 사용해 테이블의 구조를 유연하게 변경할 수 있다는 점이 흥미로웠고, 특히 제약조건(Primary Key, Not Null 등)을 다룰 때는 데이터의 무결성을 유지하는 데 왜 중요한지도 체감할 수 있었다. <br /><br />이번&nbsp;실습을&nbsp;통해&nbsp;단순한&nbsp;이론에&nbsp;머무르지&nbsp;않고,&nbsp;데이터베이스를&nbsp;실무&nbsp;관점에서&nbsp;조작하고&nbsp;관리하는&nbsp;감각을&nbsp;익힐&nbsp;수&nbsp;있었던&nbsp;점이&nbsp;가장&nbsp;큰&nbsp;수확이었고,&nbsp;앞으로&nbsp;복잡한&nbsp;데이터베이스&nbsp;환경에서도&nbsp;기본기를&nbsp;바탕으로&nbsp;유연하게&nbsp;대응할&nbsp;수&nbsp;있을&nbsp;것이라는&nbsp;자신감을&nbsp;얻을&nbsp;수&nbsp;있었다.</p>
